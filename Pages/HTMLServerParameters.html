HTML server parameters

The text citation is the only required Server.html parameter; all others are optional. Server.html parameters are separated by ampersands (&). No blanks are permitted between an & and a parameter name or around the '=' sign. Blanks are permitted within the t1...t5, fontfamily, and Englishfontfamily values. For example:
http://www.tanach.us/Server.html?Deut26:5-9&font=40&content=Vowels&Brief=1&t2=Title with blanks
The order in which parameters are added has no effect. The parameters are independent of one another. Usually, good results can be obtained with only a small number of parameters.

Parameter names and values are case-sensitive. The Server verifies parameter names, but not parameter values. For example, the pair layout=full is incorrect because of the lower case 'f' in full.

When including Server.html references in HTML documents '&' signs in the URL must be replaced by the HTML constant '&amp;'.
