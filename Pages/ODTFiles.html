ODT files

The Open Document format provides a non-proprietary full-featured, cross-platform representation of paged documents. Open Document Text (ODT) files (.odt extension) are editable by a variety of modern editors, including the free  OpenOffice.org Writer (Version 3.3+ recommended). ODT documents lack Documentary Hypothesis markings.

ODT files can be obtained in one of 4 ways:

  1. When viewing biblical text press the "ODT" button at the bottom of a text page. The file, named "Server.odt", is offered for download or opening by OpenOffice. Note the lack of an .odt extension. The ODT file will have the Layout, Content, and font characteristics of the original text in a right-justified, single-column format. Mac OS X systems will have the lineheight set automatically to 120 to compensate for the Mac rendering system.

  2. ODT files for entire biblical books can be obtained by clicking on the book name on the Home page. The middle row of the resulting page offers a table of available formats and contents. In the row labelled "ODT", click the link corresponding to the desired content ("Accents", "Vowels", "Consonants", "Morphology").

    The current format has two columns of Hebrew text per page in the "Full" Layout and "Accents" Content modes of the web site. The text is set in justified 16 point SBL Hebrew font. The 8 1/2 x 11 inch pages are setup for a double-sided printing with right side binding. The book of Genesis requires 61 pages (31 sheets) in this format when rendered on a Windows operating system. Mac OS X users will need to set the linespacing in OpenOffice Writer to compensate for the Mac rendering system (see below). The current format is experimental and subject to change. Suggestions are appreciated.

  3. ODT files can also be obtained from the Server.odt by entering a query URL into a browser address bar.
    http://www.tanach.us/Server.odt?Deut26:5-9
    See the ODTServerParameters page for a list of parameters for the Server.odt server.

  4. Zipped archives of complete Tanach books in ODT format are available in the ODTFiles directory. See the "Zipped archives of Tanach books" section of the Technical page.

ODT files allow very flexible formatting with OpenOffice.org Writer. To adjust formatting of a Tanach ODT file, open it with OpenOffice Writer and right click in the text area. Click the "Edit Paragraph Style..." menu item. The resulting window, "Paragraph Style: WLC-chapter", identifies the paragraph style and offers many tabbed options to change it. Practice changing parameters with this window.

Styles defined by the Tanach site have names starting with "WLC-"
except for the text page style, "Default".
For example, to change the line spacing, open the paragraph style window in a text region. Use the "Indents & Spacing" tab to change the "Line spacing" "Proportional" from 80% to 120%. The changes will take place immediately throughout the document when the "OK" button is pressed. To change the font size, use the "Font" tab in the paragraph window to change the "CTL font" "Size" from 16pt to 12pt. Press the "OK" button.

Operation on Mac OS X systems.

Although the ODT file format is machine-independent, the text rendering depends on the operating system. Mac OS X systems are particularly weak in their rendering of biblical Hebrew. The default formatting of ODT files is optimized for the popular Windows operating system. When the "ODT" button on a text page is pushed on Mac OS X systems, the line height parameter is automatically set to 120. Mac users viewing complete books will obtain better results by changing the font family and line spacing within OpenOffice Writer, as described above.

Obtaining customized PDF files.

ODT files are an excellent way to obtain PDF files. The OpenOffice Writer has an "Export Directly to PDF" button to do this. Be sure to wait for the page count in the lower, left corner of the Writer window to stabilize before pushing this button.

Operation with Microsoft Word 2010.

The Tanach ODT files have been validated by the former OpenOffice.org odf validator 1.0 to be in compliance with OASIS OpenDocument Format v1.2. Microsoft Word 2010 opens the Tanach ODT files; however, its rendering is misleadingly poor. Word apparently doesn't recognize the right-to-left writing mode, causing chapter numbers to appear in the wrong position. Adding the Hebrew language pack has no effect.

Microsoft Word 2010 users will have better results by copying text from a browser window and pasting it into Word. In doing so, activate the "Paragraph" styles window and, under the "Indents and Spacing" tab, set the "Direction" to "Right-to-Left". Crashes are amazingly frequent.

Printing on both sides of the paper with OpenOffice Writer.

Although the ODT files are free they're an ecologically dubious and uneconomic way to obtain large amounts of printed Tanach text.
Open Office Writer can be set to print odd and even pages separately. This allows two-sided printing with simple printers. Here's how:
  1. Activate the "Printer settings.." window under the File menu in Open Office. Press the "Options..." button on the resulting window.
  2. In the middle-left of the "Printer Options" window, click the "Left pages" and "Reversed" checkboxes only. Exit the "Printer Options..."/"Printer Settings..." windows.
  3. Press the "Print" button on the File menu.
  4. Place the resulting even pages in the printer with the blank sides and the top of the pages feeding first into the printer.
  5. Activate the "Printer Settings..."/"Printer Options..." windows in OpenOffice again.
  6. In the "Printer Options" window, click the "Right pages" checkbox only. Exit the "Printer Options..."/"Printer Settings..." windows.
  7. Press the "Print" button again.
The book of Micah is a good test case. Staple the pages together on the right side. If the total number of pages is odd, reverse the final, title page so that it appears on the outside of the stapled bundle.
